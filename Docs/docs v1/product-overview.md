# Product Overview (Terreta Hub)

## Qué es
`Terreta Hub` es un “hogar digital” para la comunidad: un lugar donde los miembros exploran recursos, se informan sobre actividades y organizan proyectos y quedadas. El producto está pensado como un punto de entrada único (“hub”) hacia experiencias como comunidad, eventos, blogs y utilidades como generador de QR.

## Para quién
- **Miembros**: personas registradas que participan activamente (navegan, guardan/crean contenido y conectan recursos internos).
- **Exploradores**: usuarios sin sesión que pueden descubrir el contenido principal y decidir si quieren ingresar.
- **Administración**: usuarios con rol admin que gestionan proyectos/recursos y contenido.

## Núcleos de experiencia actuales
- **Exploración**: landing con accesos principales (ej. “Finde en la Terreta” y “Explorar la Comunidad”).
- **Comunidad**: navegación a secciones como `Ágora`, `Miembros` y `Proyectos Destacados`.
- **Eventos**: listado y detalle de quedadas/eventos.
- **Recursos y herramientas**: utilidad como `QR` para generar enlaces internos/externos y material descargable.
- **Contenido**: `Blogs` y posts individuales.
- **Onboarding**: el flujo inicial define el estado `onboarding_completed` en el perfil.

## Problema que resuelve
En vez de tener múltiples destinos dispersos, el hub concentra:
- Encontrar qué hacer (actividades, eventos, comunidad).
- Entender quién es quién (perfiles, miembros).
- Organizar/mostrar proyectos (propuesta de valor clara).
- Facilitar acciones rápidas (por ejemplo, QR de enlaces internos para compartir).

## Alcance de `docs v1`
`docs v1` define la base conceptual y la visión de producto/UX sin asumir:
- Mobile como plataforma principal.
- Gamificación como requisito central.

En `docs v2 - mobile` tomamos esta base y la “convertimos” en un sistema de progreso, misiones y feedback diseñado para manos pulgar-centradas y sesiones cortas.

## Decisiones que buscamos mantener estables
- Claridad de navegación y consistencia de componentes.
- Feedback inmediato ante acciones importantes (guardar, crear, confirmar).
- Accesibilidad como criterio: inputs legibles, foco visible, roles/labels correctos.
- Onboarding como puente entre “conocí el hub” y “puedo hacer cosas”.
