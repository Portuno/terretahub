# Engagement Events (temporadas, retos y narrativa)

## Por qué existen
Los eventos evitan que la motivación se vuelva repetitiva. En mobile, sirven para:
- volver a activar usuarios que bajaron ritmo,
- dar “motivo” para entrar sin cambiar el core del sistema.

## Modelo de evento (recomendado)
### Evento de temporada (7-14 días)
- Objetivo: una colección narrativa (“capítulo”) + misiones cortas.
- Estructura:
  - Misiones diarias (micro-objetivos)
  - Misiones semanales (capítulo)
  - Recompensas escalonadas (participación + logro)

### Evento de mini-drop (2-4 días)
- Útil para reenganche.
- Una sola “misión grande” + 1-2 misiones pequeñas.

## Plantilla de evento (para que sea implementable)
Se recomienda que cada evento defina:
- **Nombre y tema**: la colección visual.
- **Qué cuenta**: qué acciones completan la misión.
- **Qué recompensa**: cosméticos/colecciones (y si aplica, boosters limitados).
- **Ritmo**: “lo podés completar en X minutos por día”.
- **Transparencia**: ventana temporal y reglas de elegibilidad.

## Battle Pass liviano (si se implementa)
En vez de un sistema pesado:
- usar 1-2 capas máximo,
- recompensas cosméticas por defecto,
- evitar que “comprar” sea la única vía real para completar.

## Cómo integrar con el producto actual
Ejemplos de eventos orientados al hub:
- “Semana del Descubrimiento”: misiones en `Ágora`, `Miembros` y `Proyectos`.
- “Semana de Creador”: misiones relacionadas a herramientas y publicación (ej. QR/recursos).
- “Semana de Quedadas”: participación en eventos y recordatorios.
