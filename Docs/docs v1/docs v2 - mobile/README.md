# Docs v2 - mobile

## Propósito
Guiar el diseño y desarrollo de la versión mobile con foco fuerte en gamificación con sentido: que el usuario sienta progreso, reciba feedback inmediato y encuentre motivación a través de micro-acciones reales (no grind).

Esta carpeta funciona como un “manual de diseño de gameplay” aplicada a producto: bucles, progresión, economía de recompensas, eventos y métricas.

## Qué incluye
- Estrategia mobile-first (sesiones cortas, ergonomía, accesibilidad).
- Pilares de gamificación (motivación, progreso visible, recompensas comprensibles).
- Loop principal de juego (acción -> feedback -> recompensa -> siguiente acción).
- Sistemas de progresión y retos.
- Mecánicas sociales y economía (evitando pay-to-win dañino).
- Eventos y estrategia de notificaciones.
- Métricas y anti-patrones.

## Glosario rápido
- **XP**: puntos que representan progreso (no “valor” del usuario).
- **Streak**: racha de días con actividad.
- **Misiones**: tareas cortas y claras, pensadas para el uso móvil.
- **Cosméticos**: recompensas estéticas que no cambian el poder (recomendado).
