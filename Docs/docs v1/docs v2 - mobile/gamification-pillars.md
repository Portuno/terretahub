# Gamification Pillars (con sentido)

## Pilar 1: Motivación por progreso (no por volumen)
El usuario no debería sentir que “tiene que grindear”.
Diseño recomendado:
- XP y recompensas que premian consistencia (misiones completadas, streak saludables) más que cantidad bruta.
- Progreso visible siempre: barra de nivel, misión actual, logros recientes.

## Pilar 2: Feedback inmediato y comprensible
Cada acción del usuario debe “cerrar el loop”:
- Confirmar que se completó una misión (o explicar por qué no).
- Mostrar la recompensa de forma clara y con un resumen (“ganaste XP y desbloqueaste…”) sin spam.

## Pilar 3: Misiones diseñadas para mobile
Las misiones deben ser:
- Cortas: pensadas para 3-5 minutos.
- Específicas: “hacer X” en lugar de “explorar la comunidad”.
- Repetibles sin volverse idénticas: variar el objetivo (por ejemplo, distintas secciones o distintos recursos).

## Pilar 4: Recompensas justas y no tóxicas
- Default recomendado: recompensas cosméticas (badges, marcos, temas, estilo de perfil).
- Recompensas funcionales solo si son controladas y no rompen el balance.
- Siempre explicar “por qué la ganaste” (la recompensa debe sentirse merecida).

## Pilar 5: Social con contexto (co-op y competencia sana)
Opciones:
- Desafíos por temporada (7-14 días) con objetivos acotados.
- Equipos/amigos para cooperar (misión compartida).
- Leaderboards con filtros por tiempo (“esta semana”) y reglas claras (para evitar frustración).

## Pilar 6: Economía de recompensas con guardrails
La economía debe evitar:
- Pay-to-win agresivo.
- Gráficos confusos donde el usuario no entiende el valor real de una moneda.

Regla de diseño:
- Moneda blanda para progresión del día a día.
- Moneda dura solo para cosméticos o aceleradores acotados (con límites claros).

## Pilar 7: Personalización como refuerzo identitario
Que el progreso “se vea” en el perfil:
- Cambios visuales (marco, estilo de avatar, badges).
- “Colecciones” (logros) que el usuario quiera mostrar.
