# Docs v1

## Propósito
Documentar la visión general del producto (qué es, para quién, qué resuelve) y los principios de UX que se mantienen estables mientras evolucionamos hacia versiones específicas por plataforma.

`docs v1` es el “baseline”: no asume móvil ni gamificación como requisito, solo define claridad, estructura y decisiones generales de experiencia.

## Qué incluye
- Vista de producto y estado actual.
- Principios de UX (feedback, fricción, consistencia, accesibilidad).
- Roadmap de alto nivel hacia una versión orientada a móvil y gamificación.

## Cómo usarlo
1. Leé `product-overview.md` para entender el problema y el estado actual.
2. Leé `ux-principles.md` para tomar decisiones coherentes al diseñar nuevas pantallas.
3. Usá `roadmap.md` como guía macro para no perder el rumbo entre iteraciones.
