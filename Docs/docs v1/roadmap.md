# Roadmap (v1 -> v2 mobile gamificado)

## Resumen ejecutivo
`docs v1` establece la base de producto y UX para `Terreta Hub`.

`docs v2 - mobile` traduce esa base en un sistema de juego: loop de acción corto, progresión visible (XP/niveles/récords), misiones diarias/semanales y recompensas entendibles.

## Fases

### Fase A: Loop y misiones como “core”
- Definir el loop principal: `acción del usuario -> feedback -> recompensa -> siguiente acción`.
- Diseñar misiones de baja fricción (ej. tareas cortas y claras para móvil).
- Asegurar que la recompensa llegue siempre (aunque sea pequeña) para sostener motivación.

### Fase B: Progreso visible (siente avance)
- XP con reglas consistentes (priorizar consistencia sobre volumen bruto).
- Niveles con desbloqueos visuales (badges, marcos, temas).
- Streak con protección para evitar frustración por 1 día perdido.

### Fase C: Economía responsable de recompensas
- Recompensas cosméticas como default (no pay-to-win agresivo).
- Moneda blanda para progresión saludable.
- Moneda dura solo si el producto lo requiere, con guardrails para no romper balance.

### Fase D: Eventos y social (compite o coopera)
- Eventos temporales (temporadas de 7 días) con objetivos acotados.
- Desafíos sociales (amigos/grupos) y tablas de posiciones con contexto (no solo “quién más grind”).

### Fase E: Métricas y ajustes
- Medir activación, D1/D7/D30, completitud de misiones y salud de racha.
- Ajustar dificultad y recompensas para mantener “tensión agradable” (no frustración, no aburrimiento).

## Entregables documentales esperados
- En `docs v2 - mobile`:
  - estrategia mobile-first,
  - pilares de gamificación,
  - loop,
  - progresión,
  - economía,
  - eventos/notificaciones,
  - métricas y anti-patrones.
